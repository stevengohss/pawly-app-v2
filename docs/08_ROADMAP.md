# Pawly Product Roadmap

**Document Version:** Foundation v1.2  
**Status:** Active roadmap reference

> **Revision v1.2 — 25 Jun 2026:** Removed dependency on a separate `PAWLY_APP` master file. This document now forms part of the sole `/docs` source-of-truth set.

## Purpose

This roadmap orders Pawly’s approved launch and later direction. It is intentionally outcome-based rather than calendar-based.

## Phase 0 — Product and UI Foundation

- Consolidate source-of-truth and docs.
- Establish brand assets.
- Build shared UI shell and private UI Lab.
- Establish browser preview workflow.
- Validate date input, keyboard, safe areas, and native interaction baseline.
- Build translation/personalised-copy foundation.
- Confirm reusable components before broad screen implementation.

## Phase 1 — Launch Core Loop

### Identity and entry

- Splash.
- Emotional intro.
- Pet profile.
- Basic user profile.

### Daily companion

- Today/Home.
- Daily Check-in.
- Reminders.
- Today’s Agenda.
- Notifications baseline.

### Story and sharing

- Create Moment.
- Pet Story.
- Monthly Reflection.
- Device share-sheet output.

### Community and safety

- Community Feed.
- Friends & Connections.
- Lost & Found.

### Personal space

- Profile.
- Pet profile.
- Settings.
- Drawer.

### Recognition

- Pawly Notices overlay.
- Badge/recognition area.
- Private surprise-reward/redemption capability.

## Phase 2 — Launch Stabilisation

- Improve content quality and community safety controls.
- Improve notifications.
- Verify deep links and invite recovery.
- Tune discovery/contact matching safeguards.
- Improve empty states and first-week retention.
- Test bilingual layouts and content behaviour.
- Review analytics and real user feedback.
- Prepare production/TestFlight readiness.

## Phase 3 — Meaningful Return and Belonging

- Bring Back Moments.
- This day last year / anniversaries.
- Family contributors.
- Guided Home personalisation.
- Enhanced friend/community relationship features.
- Local groups and events.
- Event verification and trusted organiser workflows.
- Expanded Pawly Notices recognition.

## Phase 4 — Care Depth

- Health records.
- Vaccinations.
- Medication.
- Weight and wellbeing trends.
- Grooming/nutrition history.
- Vet contacts and exports.

Care remains meaningful and supportive, not Pawly’s sole identity.

## Phase 5 — Local and Partner Ecosystem

- Groomer/vet/service discovery.
- Booking and appointments.
- Local recommendations.
- Partner offers.
- Pawly store.
- Reward redemption / purchase offset.
- Partner spending rewards.

## Phase 6 — Contextual AI

- Ask Pawly guidance.
- Pawly product guidance.
- Contextual place discovery.
- Safety guardrails.
- No diagnosis, dosage, or emergency-treatment replacement.

## Deferred / Not Assumed

- Facebook/Instagram friend graph import.
- Automatic social graph import.
- Wearable integration.
- Public leaderboards.
- Transactional earn-more mechanics.
- Full marketplace before core retention is proven.

## Roadmap Governance

A roadmap item moves into active implementation only after:

1. Product purpose is clear.
2. It aligns with Capture → Keep → Share → Return.
3. Source-of-truth and relevant docs are updated.
4. Technical and privacy implications are reviewed.
5. Steven approves scope.
